# 칸이 명한 순행로

몽골 제국 시대 도시 순회 최적화 데모. 고전 휴리스틱·Brute Force·양자(QAOA)를
한 Streamlit 앱에서 비교하고, 결과를 양식화된 SVG 고지도에 그린다.

## 실행 (로컬)
```
pip install -r requirements.txt
streamlit run app.py
```
Windows에서 `python`이 Store 더미면 venv 직접 사용: `.venv\Scripts\streamlit.exe run app.py`

## 배포 (Streamlit Community Cloud — 무료 고정 URL)
교내 홈페이지에 **링크로** 걸 때는 클라우드 배포가 낫다(내 PC를 켜둘 필요 없이 24시간 작동).
1. 이 폴더를 **GitHub 레포**로 푸시 (`.gitignore`가 `.venv`·`_geocache` 등 빌드 산출물을 제외 → 레포는 가볍게).
2. [share.streamlit.io](https://share.streamlit.io) → **New app** → 레포·브랜치 선택, **Main file = `app.py`**.
3. **Advanced settings → Python version = 3.12** (로컬 3.14는 클라우드 미지원, qiskit 휠 호환 위해 3.12).
4. **Deploy** → `https://<이름>.streamlit.app` 고정 URL 생성 → 교내 홈페이지에 그 링크를 건다.

배포 메모리 안전: **양자(QAOA)는 4도시까지만**(25큐비트 상태벡터는 무료 1GB 인스턴스 OOM 위험).
실용 최적화는 고전·Brute가 담당하므로 영향 없음. `순행 임무(게임 모드)`는 **베타**.
런타임은 `geo.json` 등 구워진 데이터만 쓰므로 `_geocache/`(Natural Earth·표고 원본)는 배포에 불필요 —
지형·도시를 재생성할 때만 `geobuild.py`로 다시 만든다.

## 구조
- `app.py` — Streamlit UI + 흐름 (도시 선택 → 최적화 → 지도에 주입)
- `solver.py` — 고전 TSP(NN+2-opt), Brute Force, QAOA(qiskit) — 선택적 비용행렬 `C` 인자
- `terrain.py` — 지형 이동 비용(합성). 장애는 `geo.json`에서 로드 → 대칭 비용행렬·중심거리비용
- `map_view.py` — SVG 고지도 HTML 생성 (지오 베이스맵·모드토글·백과사전 팝업·줌/팬·구간 색칠·카토그램·음영)
- `map_data.json` — 도시 75 + 보이지 않는 도시 21 (실제 좌표 투영, `geobuild.py`가 생성)
- `cities_info.json` — 도시별 권역·역사 설명(백과사전 '도시' 항목). **편집해 도시 설명 추가/수정**
- `pedia.json` — 백과사전 일반 항목(시대 배경·제국 통치 등). **편집해 항목/섹션 추가** (`sections[].entries[]`)
- `geo.json` — 실제 지리(해안선·강·호수 SVG path) + 투영된 지형 장애. `geobuild.py` 산출물
- `geobuild.py` — 오프라인 빌드: 도시 실제좌표 투영 + Natural Earth 지오 + **실제 표고(DEM)·사막·통행마스크**
  → `map_data.json`·`geo.json` 생성 (표고는 opentopodata ETOPO1, `_geocache/`에 1회 캐시)
- `opening_map.json` — 오프닝 시네마틱 전용 실제 지도(제노바~대도 해안선·노드, 별도 좌표계). `openbuild.py` 산출물
- `openbuild.py` — 오프라인 빌드: 캐시된 Natural Earth만으로 오프닝 지도 생성(`py openbuild.py`). **네트워크·메인 지도 재생성 불필요**
- `sounds/` — 효과음·배경음악(선택). `page`(책 넘기는 소리)·`bgm`·`select`·`arrive`·`star` 이름의 오디오 파일을 넣으면
  자동 인식(상단 `sounds/README.md` 참고). base64로 내장돼 별도 서버 설정 없이 동작. 우상단 🔊 버튼으로 음소거.

흐름은 Python → HTML 단방향(서버 분리 없음). 양자는 Python이 담당.
지도는 **1275년 팍스 몽골리카 75개 도시**를 실제 위경도로 투영해 양피지 화풍으로 렌더(Natural Earth, PD).

## 탐험 / 순행 그리기 (지도 상단 토글)
- **탐험**(기본): 강조·페이드·게임 패널 없는 깨끗한 지도. 도시를 클릭하면 **우측 백과사전 팝업**이
  그 도시 항목으로 열린다. 우상단 📖 버튼으로 색인(시대 배경·제국 통치·도시)을 직접 열 수도 있다.
- **순행 그리기**: 선택 도시가 강조되고 하단 게임 패널이 나타난다. 순서대로 클릭해 순행로를 작성 →
  **‘채점’을 누르면 사이드바 해법(고전/Brute/양자)이 자동 실행**되어 컴퓨터 해답과 비교된다(별도 버튼 없음).
  양자(QAOA)는 계산 폭증을 막기 위해 5개 이하에서만 동작.

## 백과사전·도시 추가 (확장 — 코드 수정·재빌드 불필요)
- **도시 설명**: `cities_info.json`에서 해당 도시의 `desc`/`region` 편집.
- **일반 항목(시대 배경·제국 등)**: `pedia.json`의 `sections[].entries[]`에 `{title, body}` 추가/수정.
  섹션 자체를 추가하려면 `sections`에 `{title, entries:[…]}`. (둘 다 새로고침이면 반영.)
- **새 도시 노드**: ① `geobuild.py` `CITIES`에 `"이름":(위도,경도,권역)` ② `cities_info.json`에
  `"이름":{region,desc}` ③ `py geobuild.py` 1회. 투영 bbox 고정이라 기존 좌표·지형 불변.
- **지형 장애**: `geobuild.py`의 `BARRIERS_LL`(이름·위도·경도·반경°·세기) 편집 후 `py geobuild.py`.

## ORBIS식 다지표 (지형 비용 ON일 때 — 순행 주체·우선순위·계절)
"거리≠시간≠비용". 사이드바에서 **순행 주체**(누가 가는가: 엘치=역참 특사·오르톡=공인 대상 길드·관료단=호구
조사관·포로단=강제 이주)·**우선순위**(최속/최저비용/최단)·**계절**(봄·여름·가을·겨울)을 고르면 **최적 경로
자체가 달라진다**: 최속은 산을 우회, 최저비용은 사막을 우회, **오르톡(낙타)은 사막을 관통하고 관료단은 크게
우회**, 겨울엔 산악·북방이 가혹(관료단처럼 악천후에 취약한 주체일수록 더). 채점 결과표에 **구간별 거리·시간·비용
3지표**가 나오고 선택한 지표 열이 강조된다.

### 역참(yam) 네트워크
지도의 **옅은 금색 점선**이 역참 본선(초원로·실크로드·볼가·루스 등)이다. **엘치(역참 특사)는 이 길 위에서만
환마로 압도적으로 빠르고**(도로 위 ~90km/일) 벗어나면 일반 기수가 된다 → 엘치 최속 경로는 역참로를 따라간다
(대도→사마르칸트 도로 점유율 97%). 낙타 상단(오르톡)은 도로에 얽매이지 않고 제 길로 사막을 건넌다.
"네트워크 위에서만 빠르다"는 ORBIS 명제의 핵심. (노선 = `geobuild.py`의 `YAM_LL` 편집 후 `py geobuild.py`.)

## 인게임 타이틀(별자리) — 스토리 / 샌드박스 선택
앱을 열면 **별자리 타이틀 화면**(밤하늘에 여정 도시들이 별처럼 빛남)이 나오고 거기서 **모드를 고른다**(일반 RPG
시작 화면처럼). 모드 선택이 **사이드바가 아니라 게임 안**에 있다 — app.py가 샌드박스·스토리 설정을 **둘 다 미리
계산해 iframe에 주입**하고, 타이틀의 두 버튼이 iframe 안에서 모드를 전환한다(단방향 흐름 유지). 좌상단 **≡ 타이틀**
버튼으로 언제든 돌아온다. 사이드바는 이제 **샌드박스 조정용**(도시·해법·지형). (샌드박스 진입 후엔 사이드바 조정이
타이틀로 끊기지 않도록 localStorage로 타이틀을 건너뛴다.)

- **🎲 샌드박스**: 별자리가 지도로 바뀐 뒤 본 게임(몽골 제국) 지도로 진입 — 도시·주체·우선순위·계절 자유.
- **📖 스토리 모드**: 하나의 이야기 **「마르코 폴로의 순행 — 칸에게 고하라」**(칼비노 '보이지 않는 도시' 모티프:
  폴로가 칸의 사신으로 제국 도시들을 돌아 **칸에게 보고**). 도시·주체(엘치)·우선순위·계절이 **고정**되고, "칸의 명"
  아래에서 순행로를 계획해 **컴퓨터 최적해 근접도로 별(★)**을 받는다(공정 채점=Brute≤9·아니면 고전).

전체 연출 흐름: **[오프닝 시네마틱] → 칸의 명(계획) → 채점 → 역참로 잇기 → 구간 미니게임 →
[도착: 도시 풍경] → … → [칸께 고하다: 보고] → 완주**.

**별자리 타이틀 → 모드 선택 → (스토리면) 오프닝 시네마틱** 흐름:
1. **별자리(밤하늘)** 타이틀 — 여정 도시 노드들이 검푸른 밤하늘에 별처럼 빛난다(지도·라벨 없음). **두 버튼(스토리/
   샌드박스)**. *제노바·베네치아는 별자리에 표시되지 않는다.*
2. **버튼 클릭** → 노드 위치 그대로 **배경이 실제 지도로 전환**(`opening_map.json`의 제노바~대도 Natural Earth
   해안선). 샌드박스면 그대로 본 게임 지도로 닫힘.
3. **스토리면** → 1초 후 **제노바가 페이드 인(펄스)**되고 *제노바를 클릭*하면 **폴로의 독백**(beat 0)이 시작된다.
   "고향을 떠올리다 →"로 카드가 닫히면 **베네치아가 활성화(페이드+펄스)**되고 *클릭*하면 **베네치아** 카드 →
   "동방으로 →" → **베네치아에서 줌아웃하며 75개 도시 노드가 드러나고**("수많은 도시" 카드) → "여정을 시작하라 →"로
   **'첫 번째 여정' 튜토리얼**: 8개 도시(에르주룸→타브리즈→헤라트→카슈가르→호탄→영하→상도→대도)가 **하나씩 떠오르며
   클릭 유도**, 각 클릭마다 **이전 지점에서 부드러운 곡선으로 이어지는 이동 애니메이션**(카라반) → **대도** 도착 →
   "순행을 시작하다 →". (튜토리얼 루트·소제목은 `scenarios.json` cities beat의 `tutorial`·`tutorial_title`로 편집.)
4. 본 게임 지도가 **대도에서 줌아웃**하며 도시 노드가 드러나고 계획이 시작된다. (beat 옵션: `focus`(카메라 대상,
   `cities`=줌아웃+75도시), `next_node`/`next_hint`(그 노드를 활성화·클릭으로 진행), `route_on_click`(클릭 후
   경로 애니).) 75개 도시 좌표는 `openbuild.py`가 `opening_map.json["cities"]`로 함께 투영한다.

**제노바·베네치아는 오프닝 전용 노드** — 별자리·본 게임(몽골 제국) 지도에는 나오지 않고 플레이어가 그쪽으로 순행할
수 없다(`opening_map.json`에만 있고 본 지도 도시 목록엔 없음). 이후 한 구간을 마치고 도시에 **도착할 때마다** 풍경
연출이, 귀환 후엔 **보고(closing)** 연출이 삽입된다. 오프닝 지도는 `py openbuild.py`로 생성(캐시된 Natural
Earth만 사용 — **네트워크·메인 지도 재생성 불필요**, 본 플레이 지도·좌표·지형 불변). 연출 텍스트·삽화는
`scenarios.json`의 `opening.beats`/`closing`, `city_scenes.json`에서 편집한다.

편집(전부 새로고침만, 재빌드 불필요): 이야기·오프닝·보고는 **`scenarios.json`**(상단 `_README` 참고 —
`opening`/`closing`/`khan_command`/`cities`/`stars`…). 도시 도착 연출은 **`city_scenes.json`**의
`scenes[도시명]`(`title`/`lines[]`/`img`). 이야기별로 도시 연출을 덮어쓰려면 `scenarios.json` 항목에
`"scenes":{"도시명":{…}}`. 파일에 없는 도시는 일반 플레이스홀더 자동 생성. **연출 텍스트·삽화(`img`)는 현재
플레이스홀더**이니 채우면 된다. scenarios에 항목을 더 넣으면 사이드바에 이야기 선택지가 생긴다(현재는 1개라 자동 시작).

## 역참로 잇기 — 2단계 미니게임 (거시→미시, 보급 사거리 퍼즐)
순행을 **채점**하면 결과창에 **"역참로 잇기 →"**가 나온다. 누르면 **전용 화면(오버레이 스테이지)**으로
전환되어, 방금 짠 거시 순회의 **각 구간(A→B)을 차례로** 푼다: 두 도시 사이에 산재한 **다수의 역참
후보(14개)** 중 **보급 사거리 안에서 가장 짧게** 잇는 **4개를 골라**(순서대로 클릭) 보급선을 만든다.
구간이 사거리(점선 원)를 넘으면 **빨간 선=보급 끊김**(감점). 컴퓨터 최적해(정확히 4 경유·각 구간 ≤R
**DP 최단**)와 비교해 **구간별 별점(★)**, 모든 구간을 마치면 **완주 요약**(구간별 별+총점)이 나오고
최고기록은 브라우저에 저장된다. 거시 단계는 종전과 동일(버튼 전엔 회귀). 역참 후보는 도시쌍 이름으로
시드된 결정적 산포라 **Python·데이터 변경 없이** 어떤 순서에도 작동한다(전부 iframe 자기완결). 미시 비용은
직선거리. 노드에 **종류(type) 필드**를 두어 향후 역참 종류 세분화(사거리·속성 차등)로 고도화할 수 있다.

## 지형 비용 (선택 — 사이드바 "지형 비용" 토글, 기본 OFF)
"연결성은 거리가 아니라 비용"(ORBIS 명제). 산맥·사막·고원을 넘는 간선을 비싸게 만들어
기하학적 직관이 빗나가게 한다. 켜면 **격자 최소비용 경로**로 풀어, 경로가 직선이 아니라
지형을 따라 고개·회랑으로 **굽고, 바다와 몽골 제국 최대영토 밖은 통행불가**라 그 안에서만 우회한다.
난이도는 **실제 표고(DEM) 경사·고도 + 실제 위치 사막**(타클라마칸·고비·카라쿰·시리아·아라비아 등)에서
