# 칸이 명한 순행로 — 프로젝트 정리 & 시연 대본

> 1275년 팍스 몽골리카를 배경으로, 순회 감찰관이 되어 **도시 순회 최적화(TSP)**를 풀고
> 인간의 직관과 컴퓨터·양자 해법을 겨루는 **교육·오락용 역사지리 시뮬레이터**.
> (기술 상세는 `HANDOFF_CLAUDE_CODE.md`, 사용법은 `README.md`.)

---

## 1. 한 줄 소개
"몽골 제국의 길 위에서, **가장 빠른 길 ≠ 가장 싼 길 ≠ 가장 짧은 길**임을 직접 풀어보는 지도 게임."

## 2. 왜 흥미로운가 — 핵심 아이디어(ORBIS 명제)
스탠퍼드 ORBIS(로마 교통망 모델)의 명제: **"연결성은 거리가 아니라 비용이다."**
- 직선으로 가까워도 산·사막이 가로막으면 *멀다*. 이 앱은 실제 표고(DEM)와 사막 분포로
  **이동 비용 지도**를 만들고, 경로가 직선이 아니라 **실제 고개·회랑을 따라 굽도록** 한다.
- 같은 두 도시라도 **누가(순행 주체)·무엇을 우선하느냐·어느 계절이냐**에 따라 최적 경로가 달라진다.
  - 예) 카슈가르→우르겐치: **낙타 상단(오르톡)은 사막을 관통(1580km)**, **관료단은 크게 우회(2288km, +45%)**.
  - 예) 바그다드→사마르칸트 최속: **엘치(역참 특사) 24일** vs **포로단(도보) 424일**.

## 3. 사용자가 보는 기능
- **양피지 고지도**: 실제 위경도로 투영한 1275년 75개 도시 + Natural Earth 해안선·강·호수.
- **두 모드 (사이드바 맨 위 토글)**
  - 🎲 **자유 순행(샌드박스)**: 도시·주체·우선순위·계절·해법을 자유로 골라 실험.
  - ⚔️ **순행 임무(게임, 베타)**: "칸의 명" 미션 — 고정 조건에서 컴퓨터 최적 대비 **별(★) 등급**.
- **백과사전(우상단 📖)**: 도시·시대 배경 항목. 탐험 모드에서 도시 클릭 → 해당 항목.
- **순행 주체(페르소나)**: 엘치(역참 특사)·오르톡(공인 대상)·관료단(호구 조사)·포로단(강제 이주)
  — 각자 속도·비용·지형 적성·계절 취약성이 다름.
- **다지표 결과표**: 채점하면 구간별 **거리·시간·비용** 3지표 + 컴퓨터 최적 대비 %.
- **해법 비교**: 고전(NN+2-opt)·Brute Force(정답)·양자(QAOA, 데모).

## 4. 기술 구조 (한눈에)
```
app.py        Streamlit UI·흐름 (도시/주체/우선순위/계절 선택 → 최적화 → 지도 주입)
solver.py     TSP: 고전(NN+2-opt)·Brute·QAOA(qiskit) — 선택적 비용행렬 C
terrain.py    지형 비용·다지표: 격자 Dijkstra 최소비용 → (비용행렬, 곡선경로, 3지표)
map_view.py   SVG 고지도 HTML (단방향 Python→HTML, 채점은 iframe 자기완결)
geobuild.py   오프라인 빌드: 실제좌표 투영 + DEM 난이도 + 통행마스크 → map_data.json·geo.json
scenarios.json 순행 임무 정의 (사용자 편집)
```
- **왜 이 구조?** 양자(qiskit)는 브라우저 불가 → Python 필요. 고지도는 미감·왜곡 제어 위해 직접 SVG.
  데이터는 **Python→HTML 단방향**(역방향 콜백 없음 → 가벼움). 과제 규모라 서버 분리(FastAPI) 안 함.

## 5. 진행 경과 (단계별)
1. **L1 기초**: 75 도시 TSP, 고전/Brute/QAOA 솔버, 플레이어 직접 그리기·채점.
2. **L2 지형 비용**: "비용 = 거리 아님" — 대칭 합성 비용 + 비용 카토그램·음영(ORBIS식).
3. **L3 실제 지리**: 실제 위경도 투영, Natural Earth 양피지 베이스맵, 백과사전 시스템,
   **실제 DEM 난이도**(경사·고도·사막) + **통행마스크**(바다·제국 밖 통행불가) + **곡선 최소비용 경로**.
4. **ORBIS 다지표**: 난이도를 산악/사막 두 성분으로 분리 → 이동수단×우선순위×계절이 다르게 가중.
5. **순행 주체(페르소나) 재구성**: 이동수단을 역사적 집단(엘치·오르톡·관료단·포로단)으로,
   계절 민감도(seasonSens) 추가 — 측정으로 경로 분기·캐릭터화 검증.
6. **파라미터 밸런싱**: 측정으로 사막 우회가 *역사적으로 옳음*을 확인(되돌리지 않음),
   기본 난이도를 4→8 도시로(인간 vs 컴퓨터 격차가 8~12에서 최대).
7. **게임 시나리오 v1(베타)**: "칸의 명" 단일 미션 + gap% 별점, localStorage 최고기록.
8. **배포 준비**: Streamlit Community Cloud용 requirements 핀·테마·.gitignore, QAOA 4도시 제한(메모리 안전).

## 7. 한계와 향후 계획
**현재 한계(정직하게)**: 비용이 대칭(방향성 없음), 역참망이 곱셈 배율로 추상화됨, 보급·위험·경제·정치
레이어 없음, 시간이 정적(여정 중 계절 고정).

**향후(현실성/재미 강화 우선순위)**
- **티어 1**: ATSP 방향 비용(강 상·하류, 고개, 계절풍) / 여정 중 계절 변화 / 위험 지표 추가.
- **티어 2**: **역참(yam) 네트워크**를 실제 간선으로(엘치는 길 위에서만 빠름) / 보급 사거리·오아시스 /
  교역 경제(상품 가치 차익 = 이윤 목표).
- **티어 3**: 정치 지도 시간축(칸국 분열·관세), 몬테카를로 위험 시뮬, 동적 이벤트,
  역사 실제 여정 시나리오(마르코 폴로·라반 바르 사우마·뤼브룩).
- **게임**: 캠페인 진행도, 페르소나별 목표 차별화(엘치=시간·오르톡=이윤·관료단=완수·포로단=생존).

## 8. 발표자료 이후의 변화 — 무엇이·왜 바뀌었나 (2026-06 대확장)

> 위 §1~§7은 **2026-06-13 발표 시점**(HANDOFF §16, "거시 TSP + 양자 데모 교육 시뮬")의 모습이다.
> 이후 §17~§26에서 프로젝트의 **무게중심이 이동**했다. 이 항목이 그 변화를 정리한다.

### 8.1 한눈에 — 중점의 이동

### 8.2 변화된 점 (기능·구현, HANDOFF §17~§26)

- **스토리 모드 일원화 + 시네마틱 오프닝**(§18·§20·§26): 인게임 **밤하늘 별자리 타이틀** → 제노바·베네치아 →
  '수많은 도시'(75 노드) → **'첫 여정' 곡선 튜토리얼** → 대도. 긴 산문은 지도를 어둡게 깐 **전면 낭독 뷰**(장 제목·삽화칸).
- **역참로 잇기 = 2단계 미니게임**(§18~§25): 거시 순회를 채점한 뒤, **각 구간의 보급선을 직접 잇는다.**
  - **종류 3종**(§19): 모린 얌(말·멀리·비쌈)·테르겐 얌(수레·가깝·저렴)·나린 얌(밀행·중간) + **주체 적성**(엘치→모린 등).
  - **최소 비용 가변 보급로**(§22): 고정 개수(4개) 강제를 **폐기** → "닿는 역참을 몇 개든 이어 **총비용 최소화**"(Dijkstra 최적 대비 별).
  - **실제 지형 회랑**(§21·§23): 전용 스테이지에 본 지도 슬라이스를 변환해 그려 — 실제 방향·해안선·강·역참로 위에서 플레이.
  - **역참 수 = 실제 거리 비례**(§23-추가): 초장거리는 더 많은 역참(소프트 캡 ~11) → 역참 총비용이 **투어 길이를 추종**(두 층 강화).
  - **지형 형성 보급**(§23-추가2): 사막에선 **실효 사거리↓**, 사이트는 **봉우리·사막내부 회피·강/오아시스 선호** → 실크로드식 림 경로가 규칙에서 *발생*.
  - **온보딩·완성도**(§24·§25): 첫 구간 **튜토리얼**(?로 재확인) · **정직한 보급 링**(effReach 블롭) · 별점/UX 보정.
- **그 밖에**: 백과사전 1275 콘텐츠(§17) · 효과음/BGM 시스템(§18) · **사이드바 초기 접힘** · 하단 **도움말 보강**.

### 8.3 중점의 변화 — 왜 이렇게 이동했나 (분석)

1. **단일 TSP → 이중 최적화.** 거시 순회만으론 "한 번 풀면 끝"이었다. 각 구간을 *어떻게 보급하느냐*(역참로)를 미시 퍼즐로
   더해, **두 층이 서로 강화**된다(좋은 순서 = 짧은 구간 = 싼 보급). 학술적으로도 **두 고전 최적화**(순회 TSP + 최소비용 경로)를
   한 지도에서 대비시키는 깔끔한 짝이 됐다.
2. **데모 → 게임·몰입.** 3분 시연을 넘어 스토리·연출·튜토리얼로 *플레이*되게 했다. 마르코 폴로–칼비노 서사가 "왜 도시를
   순회하는가"의 동기를 부여한다.
3. **역참의 실현.** 발표 때 '미래 티어 2'였던 역참망·보급 사거리·오아시스가 **핵심 메커니즘**이 됐다. 역사적 야미(Örtöö)
   기능 분류(모린/테르겐/나린)를 그대로 게임 규칙으로 옮겼다.
4. **양자에서 '두 문제의 대비'로.** QAOA는 거시 TSP의 한 면으로 남되, 미시는 **굳이 양자로 끼우지 않고 Dijkstra**로 둬
   "NP-난해 순회 vs 다항시간 최단경로"라는 교육적 대비를 택했다(정직성).
5. **현실성 ↔ 게임성 동시.** 실제 km·지형이 규칙에서 림 경로를 *발생*시키되(억지 배치 아님), 정직한 링·소프트 캡·튜토리얼로
   플레이성을 확보했다. 모든 밸런스는 **측정(preview E2E)으로 검증** 후 반영.

### 8.4 시연 대본 — 갱신 포인트 (§6은 거시 단계까지만 다룸)

발표 시 **거시 채점 다음에 미시 단계**를 잇는다:
1. **[스토리 진입]** 별자리 타이틀 → 오프닝 낭독("적히지 않은 도시"). 2. **[거시 순행]** 도시 순서대로 → 채점·별.
3. **["역참로 잇기"]** "이제 순서를 푸는 데서 끝나지 않습니다. **각 길을 어떻게 보급하느냐**까지 — 닿는 역참을 이어
